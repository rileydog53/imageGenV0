## ⚠ Rendering Bugs Discovered in Live Testing (cisplatin `cellular_schematic`, 2026-05-23)

Rendered a 7-entity, 4-compartment `cellular_schematic` figure (Cisplatin → Passive Diffusion → Activated Cisplatin → Pt–DNA Adducts → Replication Block → Apoptosis → Cell Lysis). Four concrete defects observed. Items L15–L19 below track fixes; each entry includes the exact file/function and the change needed.

| Defect | Screenshot observation | Root cause | Fix item |
|--------|----------------------|------------|----------|
| Entity bbox overflow | "Activated Cisplatin" and "N7 crosslink" label cut off at right canvas edge | `_graph_positions` places entity **centers** anywhere in [0, canvas_w]; half-bbox width overflows SVG viewport | **L15** |
| Arrow label overflow | Arrow label text near canvas edge is clipped | `place_labels` has no canvas-bounds check; candidates outside [0,W]×[0,H] are scored the same as in-bounds ones | **L18** |
| U-shaped linear chain | Cisplatin ends up top-right, Cell Lysis top-left — spring layout ignores DAG direction | `nx.spring_layout` has no concept of topological order; all-zero initial positions let it settle in any orientation | **L16** |
| Cross-compartment arrows | Arrows between different bands are straight lines that cross band label text and borders | Arrow routing does not consult compartment membership; no waypoints are injected at band boundaries | **L17** |
| Empty canvas bottom | Verifier flags `needs_crop=True`; bottom ~250 px of SVG is white | `compute_pathway_canvas` clamps to `max(sum_dynamic, 600)`; when bands sum to <600 the extra whitespace is dead | **L19** |

## ⚠ Rendering Bugs Discovered in Live Testing (10-figure variety sweep, 2026-05-25)

Ran a 10-prompt variety sweep across all archetypes (single-object → 8+ entity hub; chemistry, signalling, workflow). **Succeeded:** figures that fit the compartment/linear grammar — GPCR-in-bilayer, glucose ring-closure, RTK activation. **Failed or degraded:** anything needing branching, convergence, multi-step chemistry, or many wide nodes. Six distinct defects; fix items L20–L24, R6, V1 below.

| Defect | Observation | Root cause | Fix item |
|--------|-------------|------------|----------|
| Reaction renders as text boxes, not structures | Esterification & SN2 drew labelled boxes; SMILES ignored; semantic FAIL (`reaction_0` missing) | A species both consumed *and* produced trips `_is_multistep_reaction`; `render_figure` silently coerces archetype to PATHWAY, which never calls the SMILES renderer. Verifier reads the original `reaction_scheme` IR and looks for `reaction_0` → false FAIL | **R6**, **V1** |
| Branch / hub topology collapses to one row | G6P branch point & 4→1→3 signalling hub drew a flat row; convergence lost, NF-κB orphaned with a giant arrow | Entity y = compartment band centre (`_compute_bands` + `_graph_positions`); no compartments → one implicit band → every entity shares one y, can only fan horizontally | **L20** |
| Wide figures clipped at edges | EGFR, "Pentose Phosphate Pa…", "Tissue Sample", "GPCR"→"PCR" cut off | `compute_pathway_canvas` fixes width at the 800px floor; width never grows with entity count or label length | **L21** |
| Whitespace never trimmed | Every figure flagged `needs_crop=True` | `legibility_check` computes the flag but no step consumes it — the docstring's "next step (intelligent zoom/crop)" is unimplemented | **L22** |
| Cascade renders backwards | MAPK drew EGFR→ERK right-to-left | Feedback edge ERK⊣RAF makes the graph cyclic; `nx.topological_generations` can't rank a cycle, so L16's x-seeding falls back to unconstrained spring | **L23** |
| Arrow labels overlap node text | "at Ser", "feedback", "allosteric", "QC check", all SN2 labels collide | L2's ladder degrades gracefully to flagged overlap, but reserves no corridor for edge labels on short arrows beside wide nodes | **L24** |

**Skill-authoring guidance (SKILL.md prompt, not engine code) — ✓ folded into `~/.claude/skills/imageGen/SKILL.md` (2026-05-26), "Encoding pitfalls" section + archetype note:**
- Encode reactions so no species is both source *and* target. "A + B → C + D" = parallel reactant→product edges (e.g. `A→C, B→D`), **not** a chain (`A→C, B→C, C→D`). The chain creates a false intermediate and triggers the R6/V1 failure.
- Use `mechanism_cartoon` for reaction mechanisms, not `reaction_scheme`.
- Don't model decorations (phosphosites, N-/C-termini) as separate entity nodes — they belong as glyphs on the relation (as the working RTK figure did with its P-marker).
